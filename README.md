## فایل‌های پروژه

- `matrix_library.h`   : فایل هدر شامل اعلان کلاس‌ها و توابع
- `matrix_library.cpp` : پیاده‌سازی کامل کلاس‌ها و توابع
- `demonstration.cpp`  : برنامه نمایشی که گزارش HTML تولید می‌کند
- `report_sample.html` : نمونه خروجی برنامه (تولید شده توسط demonstration.cpp)
- `test_cases.txt`     : نمونه‌های تست برای عملیات مختلف
- `README.md`          : این فایل مستندات

## نحوه کامپایل و اجرا

### 1. کامپایل فایل‌ها با کامپایلر g++

```bash
g++ -std=c++11 matrix_library.cpp demonstration.cpp -o matrix_demo
```

### 2. اجرای برنامه

```bash
./matrix_demo
```

### 3. پس از اجرا، فایل‌های زیر تولید می‌شوند:

- `report_sample.html` : گزارش کامل عملیات با فرمت HTML
- `test_cases.txt`     : نمونه‌های تست برای اعتبارسنجی

---

## محدودیت‌های پیاده‌سازی

1. **دترمینان**: محاسبه بازگشتی فقط تا ماتریس 4×4 پشتیبانی می‌شود (جلوگیری از سرریز پشته)
2. **ضرب خارجی**: فقط برای بردارهای دقیقاً سه‌بعدی پیاده‌سازی شده است
3. **ماتریس معکوس**: فقط برای ماتریس‌های مربعی و معکوس‌پذیر (دترمینان ≠ 0)
4. **دقت اعشاری**: نمایش ماتریس‌ها با 2 رقم، نتایج عددی با 4 رقم اعشار

---

## ویژگی‌های گزارش HTML

- استفاده از تگ‌های معنایی HTML5 (`header`, `section`, `footer`)
- طراحی واکنش‌گرا برای نمایش مناسب در موبایل و دسکتاپ
- تمایز واضح بین ورودی‌ها (پس‌زمینه آبی کمرنگ) و خروجی‌ها (پس‌زمینه سبز کمرنگ)
- برچسب‌گذاری ابعاد برای تمام ماتریس‌ها و بردارها در کنار هر ماتریس/بردار
- استفاده از نمادهای ریاضی استاندارد (× برای ضرب ماتریسی، · برای ضرب داخلی)
- نمایش پیام‌های خطا با بک‌گراند قرمز و آیکون ❌ برای جلب توجه
- پشتیبانی کامل از زبان فارسی با جهت‌دهی راست به چپ (`dir="rtl"`)

## انتخاب‌های پیاده‌سازی مهم

1. استفاده از کلاس‌های جداگانه `Matrix` و `Vector` برای رعایت اصول شیءگرا
2. پیاده‌سازی صریح دترمینان 2×2 و 3×3 برای بهبود عملکرد (سریع‌تر از بازگشتی)
3. مدیریت خطا با پیام‌های فارسی دقیق شامل ابعاد و شاخص‌های خطا
4. تولید گزارش با استایل داخلی (Inline CSS) برای استقلال کامل فایل HTML
5. استفاده از رفرنس‌ها برای جلوگیری از کپی غیرضروری داده‌ها در پارامترها
